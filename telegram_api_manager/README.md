# Telegram API Manager

Десктопное приложение на Python с графическим интерфейсом (Tkinter) для получения **API ID** и **API Hash** через [my.telegram.org](https://my.telegram.org) и локального хранения профилей с этими данными.

Весь процесс авторизации и создания приложения выполняется в одном окне — без ручного копирования из браузера.

## Возможности

- **Получить API** — пошаговый мастер:
  1. Ввод номера телефона (международный формат, например `+79001234567`)
  2. Подтверждение кодом из Telegram
  3. Создание приложения или чтение уже существующих `api_id` / `api_hash`
  4. Копирование результата и сохранение в менеджер
- **Менеджер пользователей** — добавление, редактирование, удаление профилей; экспорт выбранного профиля в JSON в буфер обмена
- Локальное хранение в файле `users_data.json`
- Запуск одним кликом через `run.bat` (создаёт `.venv` и ставит зависимости при первом запуске)

## Требования

- Windows (приложение ориентировано на `run.bat`; на Linux/macOS можно запускать вручную через `python main.py`)
- Python 3.10+ ([python.org](https://www.python.org/))
- Доступ в интернет и активный аккаунт Telegram на указанном номере
- Tkinter (обычно входит в стандартную установку Python на Windows)

## Быстрый старт

1. Откройте папку `telegram_api_manager`.
2. Запустите **`run.bat`** — при первом запуске создастся виртуальное окружение `.venv` и установятся зависимости из `requirements.txt`.
3. На вкладке **«Получить API»** пройдите шаги мастера.
4. Сохраните профиль на шаге 4 или на вкладке **«Менеджер пользователей»**.

### Ручная установка

```bash
cd telegram_api_manager
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python main.py
```

## Структура проекта

| Файл | Назначение |
|------|------------|
| `main.py` | GUI: мастер получения API и менеджер профилей |
| `mytelegram_client.py` | HTTP-клиент для my.telegram.org |
| `user_storage.py` | Чтение и запись `users_data.json` |
| `users_data.json` | Локальная база профилей (создаётся/обновляется приложением) |
| `requirements.txt` | Зависимости: `requests`, `beautifulsoup4` |
| `run.bat` | Запуск с автонастройкой окружения |
| `error.log` | Трассировка при критической ошибке запуска (если возникнет) |

## Формат `users_data.json`

```json
{
  "users": [
    {
      "id": "uuid",
      "username": "my_account",
      "phone": "+79001234567",
      "api_id": "12345678",
      "api_hash": "abcdef0123456789abcdef0123456789",
      "app_title": "My App",
      "app_shortname": "myapp",
      "notes": "",
      "created_at": "2026-05-16T12:00:00+03:00",
      "updated_at": "2026-05-16T12:00:00+03:00"
    }
  ]
}
```

Поле `username` — произвольное имя профиля в менеджере (не обязательно совпадает с @username в Telegram).

## Безопасность

- **`api_hash` — секрет.** Не публикуйте его, не коммитьте `users_data.json` в публичные репозитории и не передавайте третьим лицам.
- Данные хранятся **только на вашем компьютере** в открытом виде в JSON.
- При сбросе мастера (**«Новый запрос»**) сессия на my.telegram.org сбрасывается локально (`logout`).

Рекомендуется добавить в `.gitignore`:

```
users_data.json
error.log
.venv/
```

## Устранение неполадок

| Проблема | Что проверить |
|----------|----------------|
| «Не найден Python» при запуске `run.bat` | Установите Python и отметьте «Add to PATH» в установщике |
| Окно не открывается | Откройте `error.log` в папке проекта |
| Ошибка сети | Прокси, VPN, доступ к `my.telegram.org` |
| «Не найден hash формы» | Сайт my.telegram.org мог изменить вёрстку — потребуется обновление `mytelegram_client.py` |
| Код не приходит | Номер в формате `+код_страны...`, код смотрите в приложении Telegram |

## Зависимости

- [requests](https://pypi.org/project/requests/) — HTTP-запросы к my.telegram.org
- [beautifulsoup4](https://pypi.org/project/beautifulsoup4/) — разбор HTML страницы приложений

## Ограничения

- Работа завязана на текущую структуру сайта [my.telegram.org](https://my.telegram.org); при изменениях на стороне Telegram парсер может потребовать доработки.
- Приложение предназначено для **личного** получения API-ключей для своих проектов (боты, клиенты Telethon/Pyrogram и т.д.). Соблюдайте [условия использования Telegram](https://telegram.org/tos).
